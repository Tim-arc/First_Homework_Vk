# Проект AskPupkin (Q&A Платформа)

Это учебный проект, представляющий собой сайт-аналог Stack Overflow, разработанный на фреймворке Django.

## Основные технологии

*   **Бэкенд:** Python 3.10+, Django
*   **База данных:** PostgreSQL
*   **Окружение:** Docker (для базы данных)

## Предварительные требования

Перед началом работы убедитесь, что на вашей машине установлены:

*   Python (версия 3.10 или выше)
*   `pip` (менеджер пакетов Python)
*   `virtualenv` (для создания изолированных окружений)
*   Docker
*   Docker Compose

## Установка и запуск проекта

**1. Клонирование репозитория**

```bash
git clone https://github.com/Tim-arc/First_Homework_Vk.git
```

**2. Создание и активация виртуального окружения**

```bash
# Создаем окружение в папке .venv
python -m venv .venv

# Активируем его (для macOS/Linux)
source .venv/bin/activate

# Для Windows
.venv\Scripts\activate```

**3. Установка зависимостей Python**

```bash
pip install -r requirements.txt```

**4. Запуск базы данных в Docker**

Этот шаг запустит контейнер с PostgreSQL в фоновом режиме.

```bash
docker-compose up -d
```
*   `up` - создает и запускает контейнеры.
*   `-d` (detached) - запускает контейнеры в фоновом режиме.
*   При первом запуске Docker скачает образ PostgreSQL, это может занять несколько минут.

**5. Применение миграций**

Эта команда создаст все необходимые таблицы в базе данных, которая теперь работает внутри Docker.

```bash
python manage.py migrate
```

**6. Создание суперпользователя**

Это необходимо для доступа к админ-панели Django (`/admin/`).

```bash
python manage.py createsuperuser
```
Следуйте инструкциям в терминале, чтобы задать логин, email и пароль.

**7. Наполнение базы тестовыми данными (опционально)**

Вы можете наполнить базу большим количеством тестовых данных с помощью нашей management-команды.

```bash
# Создаст 10 пользователей, 100 вопросов, 1000 ответов и т.д.
python manage.py fill_db 10
```

**8. Запуск сервера для разработки**

```bash
python manage.py runserver
```
Теперь сайт доступен в вашем браузере по адресу [http://127.0.0.1:8000/](http://127.0.0.1:8000/).

## Полезные команды Docker

*   **Остановить контейнер с базой данных:**
    ```bash
    docker-compose down
    ```
*   **Посмотреть логи контейнера (если что-то пошло не так):**
    ```bash
    docker-compose logs db
    ```
